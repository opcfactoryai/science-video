# 贡献指南

感谢你考虑为本项目贡献代码！以下是一些基本规范。

## 分支命名

- `feature/<简短描述>` — 新功能
- `fix/<简短描述>` — Bug 修复
- `test/<简短描述>` — 测试相关

## 提交规范

```
<type>: <简短描述>

<详细说明（可选）>
```

类型包括：
- `feat` — 新功能
- `fix` — Bug 修复
- `refactor` — 重构
- `docs` — 文档
- `test` — 测试
- `chore` — 杂项

## PR 流程

1. 在 dev 或 feature 分支开发
2. 提交并推送到远程
3. 创建 Pull Request，关联对应 Issue
4. 等待 Review 通过后合并

## 代码风格

- 遵循项目现有的代码风格
- 提交前确保无报错
- 保持注释简洁明了
